# AWS-Static-Website-Deployment
# AWS S3 Static Website with Terraform & GitLab CI/CD

## Overview

This project demonstrates how to deploy and manage a static website on AWS using Amazon S3, Terraform Infrastructure as Code (IaC), and GitLab CI/CD pipelines.

The goal of this lab was to automate infrastructure deployment, apply Infrastructure as Code best practices, and gain hands-on experience with cloud automation workflows.

---

## Technologies Used

* AWS S3
* Terraform
* GitLab CI/CD
* Git
* Infrastructure as Code (IaC)
* AWS IAM

---

## Architecture

```text
Developer
    │
    ▼
GitLab Repository
    │
    ▼
GitLab CI/CD Pipeline
    │
    ▼
Terraform
    │
    ▼
Amazon S3 Bucket
    │
    ▼
Static Website Hosting
```

---

## Objectives

* Deploy a static website utilizing Amazon S3
* Provision infrastructure using Terraform
* Create reusable Terraform variables and outputs
* Implement GitLab CI/CD automation
* Apply Infrastructure as Code best practices
* Utilize Git branching and version control workflows

---

## Key Features

### Infrastructure as Code

* Provisioned AWS resources through Terraform
* Utilized variables, outputs, and state management
* Created reusable and maintainable infrastructure

### CI/CD Automation

* Automated Terraform validation
* Automated Terraform planning
* Automated deployment workflows
* Implemented deployment approval processes

### Version Control

* Managed infrastructure through Git
* Utilized feature branches and merge workflows
* Maintained deployment consistency through source control
